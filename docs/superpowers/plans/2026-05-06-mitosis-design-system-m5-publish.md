# Mitosis Design System — M5 Publish v1.0.0 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Publish **10 npm packages** at v1.0.0 (`@revue-de-presse/design-system-{react,vue,svelte,solid,lit,angular,qwik,preact,stencil,alpine}`) with their respective Provider sub-packages, deploy the Histoire docs site to GitHub Pages, configure changesets so future releases require an opt-in changelog entry, and add per-target READMEs so each npm page is consumable.

**Architecture:** Each target's `output/<target>/` becomes a publishable package via a small `package.json` template + `tsup` build emitting `dist/`. Changesets (`@changesets/cli`) drives version bumps. GitHub Actions workflow handles publish on tagged releases. Histoire's static build deploys via `actions/deploy-pages`.

**Tech Stack:** `@changesets/cli`, `tsup` (already in M1), `actions/deploy-pages`, `actions/upload-pages-artifact`.

**Prerequisite:** M1, M2, M3, M4 plans complete and merged. 27 components emit cleanly to all 10 targets. Visual regression green on Vue + React. CI green.

**Companion spec:** `docs/superpowers/specs/2026-05-06-mitosis-design-system-design.md` §3.1 (ten npm packages, all sharing the same version) and §9 M5.

## Task 2: Per-target manifests via a script

**Files:**
- Create: `design-system/scripts/prepare-publish.mjs`

The script generates one `package.json` and one `README.md` per `output/<target>/` directory from a small in-script template. Re-running it is idempotent.

- [ ] **Step 1: Write the script**

Write `design-system/scripts/prepare-publish.mjs`:

```js
#!/usr/bin/env node
import { writeFileSync, readFileSync, existsSync, mkdirSync } from 'node:fs';
import { join, dirname } from 'node:path';
import { fileURLToPath } from 'node:url';

const __dirname = dirname(fileURLToPath(import.meta.url));
const root = join(__dirname, '..');
const ROOT_PKG = JSON.parse(readFileSync(join(root, 'package.json'), 'utf8'));
const VERSION = ROOT_PKG.version || '0.0.0';

const TARGETS = [
  { id: 'react',   peer: { react: '>=18' } },
  { id: 'vue',     peer: { vue: '>=3' } },
  { id: 'svelte',  peer: { svelte: '>=4' } },
  { id: 'solid',   peer: { 'solid-js': '>=1' } },
  { id: 'lit',     peer: { lit: '>=3' } },
  { id: 'angular', peer: { '@angular/core': '>=18', '@angular/common': '>=18' } },
  { id: 'qwik',    peer: { '@builder.io/qwik': '>=1.5' } },
  { id: 'preact',  peer: { preact: '>=10' } },
  { id: 'stencil', peer: { '@stencil/core': '>=4' } },
  { id: 'alpine',  peer: { alpinejs: '>=3' } },
];

for (const target of TARGETS) {
  const dir = join(root, 'output', target.id);
  if (!existsSync(dir)) {
    console.warn(`skipping ${target.id}: directory missing — has Mitosis emitted?`);
    continue;
  }
  const distDir = join(dir, 'dist');
  if (!existsSync(distDir)) mkdirSync(distDir, { recursive: true });

  const pkg = {
    name: `@revue-de-presse/design-system-${target.id}`,
    version: VERSION,
    description: `Revue de presse design system — ${target.id} build`,
    license: 'MIT',
    type: 'module',
    main: './dist/index.js',
    module: './dist/index.js',
    types: './dist/index.d.ts',
    exports: {
      '.': { import: './dist/index.js', types: './dist/index.d.ts' },
      './tokens.css': './dist/tokens.css',
      './sprite.svg': './dist/sprite.svg',
      './package.json': './package.json',
    },
    files: ['dist'],
    sideEffects: ['*.css', '*.svg'],
    repository: {
      type: 'git',
      url: 'https://github.com/revue-de-presse/design-system.git',
      directory: `output/${target.id}`,
    },
    publishConfig: { access: 'public' },
    peerDependencies: target.peer,
  };

  writeFileSync(join(dir, 'package.json'), JSON.stringify(pkg, null, 2) + '\n');

  const readme = `# @revue-de-presse/design-system-${target.id}

Revue de presse design system, ${target.id} build.

## Install

\`\`\`bash
${packageManagerInstall(target.id)}
\`\`\`

## Use

Import the components and the design tokens once at the root of your app:

\`\`\`${importExample(target.id)}
\`\`\`

The icon sprite must be mounted once per page (it powers \`<Icon name="…" />\`). See the source repo for the full list of glyphs.

## Documentation

Component docs at <https://revue-de-presse.github.io/design-system/>.

Spec and source: <https://github.com/revue-de-presse/design-system>.

## License

MIT.
`;

  writeFileSync(join(dir, 'README.md'), readme);
  console.log(`prepared output/${target.id}/{package.json, README.md}`);
}

function packageManagerInstall(t) {
  return `pnpm add @revue-de-presse/design-system-${t}`;
}

function importExample(t) {
  switch (t) {
    case 'vue':
      return `ts
import 'YOUR_NODE_MODULES/@revue-de-presse/design-system-vue/dist/tokens.css';
import { Button, Calendar, BlueskyPostCard } from '@revue-de-presse/design-system-vue';`;
    case 'react':
      return `tsx
import '@revue-de-presse/design-system-react/dist/tokens.css';
import { Button, Calendar, BlueskyPostCard } from '@revue-de-presse/design-system-react';`;
    case 'lit':
    case 'stencil':
      return `html
<link rel="stylesheet" href="/node_modules/@revue-de-presse/design-system-${t}/dist/tokens.css">
<script type="module" src="/node_modules/@revue-de-presse/design-system-${t}/dist/index.js"></script>`;
    case 'alpine':
      return `html
<link rel="stylesheet" href="/node_modules/@revue-de-presse/design-system-alpine/dist/tokens.css">
<script type="module" src="/node_modules/@revue-de-presse/design-system-alpine/dist/index.js"></script>`;
    default:
      return `ts
import '@revue-de-presse/design-system-${t}/dist/tokens.css';
import { Button } from '@revue-de-presse/design-system-${t}';`;
  }
}
```

## Task 3: Per-target tsup builds

**Files:**
- Create: one `tsup.config.ts` per target under `output/<target>/`
- Create: `design-system/scripts/build-all-targets.mjs`

The Mitosis emit gives us source files; tsup compiles them to `dist/index.js` + `dist/index.d.ts` + bundles. Each target's tsup config is a tiny file. The orchestrator script runs them all.

- [ ] **Step 1: Write the orchestrator**

Write `design-system/scripts/build-all-targets.mjs`:

```js
#!/usr/bin/env node
import { execSync } from 'node:child_process';
import { existsSync, copyFileSync } from 'node:fs';
import { join, dirname } from 'node:path';
import { fileURLToPath } from 'node:url';

const __dirname = dirname(fileURLToPath(import.meta.url));
const root = join(__dirname, '..');

const TARGETS = ['react', 'vue', 'svelte', 'solid', 'lit', 'angular', 'qwik', 'preact', 'stencil', 'alpine'];

for (const t of TARGETS) {
  const dir = join(root, 'output', t);
  if (!existsSync(dir)) {
    console.warn(`skipping ${t}: missing emit`);
    continue;
  }
  const cfg = join(dir, 'tsup.config.ts');
  if (!existsSync(cfg)) {
    console.warn(`skipping ${t}: missing tsup config`);
    continue;
  }
  console.log(`▶ building ${t}`);
  execSync(`pnpm exec tsup --config ${cfg}`, { stdio: 'inherit', cwd: root });
  // Co-locate tokens + sprite for each package's dist
  copyFileSync(join(root, 'src/tokens/tokens.css'), join(dir, 'dist/tokens.css'));
  copyFileSync(join(root, 'src/icons/sprite.svg'), join(dir, 'dist/sprite.svg'));
  console.log(`✓ built ${t}`);
}
```

## Task 5: Release workflow

**Files:**
- Modify: `design-system/.github/workflows/ci.yml` — add the release job

- [ ] **Step 1: Add a release job**

Append to `design-system/.github/workflows/ci.yml`:

```yaml
  release:
    name: release
    runs-on: ubuntu-latest
    needs: [build-and-test, visual]
    if: github.ref == 'refs/heads/main' && github.event_name == 'push'
    permissions:
      contents: write
      pull-requests: write
      id-token: write
    defaults:
      run:
        working-directory: design-system
    steps:
      - uses: actions/checkout@v4
        with: { fetch-depth: 0 }
      - uses: pnpm/action-setup@v4
        with: { version: 9 }
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: pnpm
          cache-dependency-path: design-system/pnpm-lock.yaml
          registry-url: 'https://registry.npmjs.org'
      - run: pnpm install --frozen-lockfile
      - run: pnpm build:mitosis
      - run: pnpm prepare-publish
      - run: pnpm build:targets
      - name: Create release PR or publish
        uses: changesets/action@v1
        with:
          version: pnpm version-packages
          publish: pnpm release
          createGithubReleases: true
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          NPM_TOKEN: ${{ secrets.NPM_TOKEN }}
```

This uses the standard changesets action: when the queue holds pending changesets it opens a "Version packages" PR. When that PR is merged, it publishes to npm.
